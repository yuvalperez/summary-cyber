מתי שאפלקציה פגועה לSQL injection ותוצאות הquery מוחזרות בתגובה של האפלקציה ניתן להשתמש במילת המפתח UNION כדאי לקבל נתונים על טבלות אחרות מתוך הdatabase.
וזה מאפשר לנו לבצע ה `SELECT queries` נוספת אחת או יותר ולצרף את התוצאות לqueries המקורית.
למשל:
`SELECT a, b FROM table1 UNION SELECT c, d FROM table2` 
הSQL query מחזירה לנו  תוצאה אחת עם שתי עמודות המכילה את a, 
וb מטבלה אחת עמוד c  , וd מטבלה 2.
בשביל ש `UNION` query יעבוד צריך שתי דברים: 

1. הquery בודדות חייבות להחזיר את אותו מספר של עמודות.
2. סוגי הנתונים בכל עמודה חייבים להיות תואמים בין השאילתות השונות.

ובשביל שהמתקפה תעבוד צריך שתי דברים שצריך למצוא :
1. כמה עמודות מוחזרות הquerys המקורית.
2. אילו עמודות מחזירות את הquerys המקורית הן כמה סוגי נתונים המתאמים להכיל את התוצאות הquery שהוזרקה.
### שיטות למציאת מספר העמודות (Column Count) #SQLi-Discovery

כדי שמתקפת `UNION` תצליח, אנחנו חייבים לדעת בדיוק כמה עמודות יש בשאילתה המקורית. יש שתי דרכים עיקריות לעשות זאת:
#### שיטה 1: שימוש ב-ORDER BY

השיטה הזו מתבססת על פקודת המיון של SQL. אנחנו מבקשים מהשרת למיין לפי אינדקס של עמודה (1, 2, 3...).

- **הלוגיקה:** אם נבקש למיין לפי עמודה מספר 3, והשרת מחזיר רק 2 עמודות, הוא יזרוק שגיאה.
 
- **הPayloads:** `' ORDER BY 1--` `' ORDER BY 2--` `' ORDER BY 3--`

- **התוצאה:** ברגע שמופיעה שגיאה (Error 500, הודעת "Out of range", או שהתוכן נעלם), סימן שהגענו למספר גבוה מדי. אם ב-3 קיבלנו שגיאה, סימן שיש **2 עמודות**.

#### שיטה 2: שימוש ב-UNION SELECT NULL

השיטה הזו מזריקה שאילתה חדשה ומנסה להתאים אותה למקורית בעזרת ערכי `NULL`.

- **למה NULL?** כי `NULL` מתאים לכל סוג נתונים (טקסט, מספר, תאריך). זה חוסך לנו שגיאות של "סוג נתונים לא תואם".
   
- **הPayloads:** `' UNION SELECT NULL--` `' UNION SELECT NULL,NULL--` `' UNION SELECT NULL,NULL,NULL--`
 
- **התוצאה:** אם מספר ה-`NULL`-ים לא תואם למספר העמודות המקורי, השרת יחזיר שגיאה. ברגע שהבקשה עוברת בהצלחה (לפעמים תראה שורה ריקה נוספת באתר), מצאת את המספר הנכון.



###  מציאת עמודה עם סוג נתונים מתאים (Data Types)

אחרי שמצאנו שיש למשל 2 עמודות, אנחנו צריכים לדעת איזה מהן יכולה להכיל **טקסט** (כדי שנוכל להוציא שמות משתמשים וסיסמאות).

- **איך בודקים?** מחליפים כל `NULL` בתורו במחרוזת בדיקה (כמו `'a'`).

- **הPayloads:**
 
  1. `' UNION SELECT 'a', NULL--` (אם יש שגיאה, עמודה 1 היא כנראה מספר).

 2. `' UNION SELECT NULL, 'a'--` (אם זה עובד, עמודה 2 יכולה להכיל טקסט).

## Database-specific syntax: #Database-specific-syntax
בOracle כל `SELECT`query חייב להשתמש ב`FROM` ולציין טבלה. יש בתוף oracle טבלה מובנת שקוראים לה  `dual`. ברוב מסדי הנתונים (כמו MySQL או PostgreSQL), אפשר לכתוב שאילתה פשוטה כזו: `SELECT NULL, NULL` והשרת פשוט יחזיר לך שורה עם שני ערכים ריקים.

**אבל ב-Oracle זה לא עובד.** אצלם כל `SELECT` חייב להסתיים במילת המפתח `FROM` ושם של טבלה קיימת. כדי לפתור את זה בלי לנחש שמות של טבלאות, Oracle יצרו טבלת "דמה" שקיימת בכל מערכת וקוראים לה **`DUAL`**.

- **הזרקה רגילה:** `' UNION SELECT NULL, NULL--` (תיכשל ב-Oracle).

- **הזרקה ל-Oracle:** `' UNION SELECT NULL, NULL FROM DUAL--` (תעבוד).
### הבעיה של MySQL: המקפים והרווח

כדי שההזרקה תעבוד, אנחנו צריכים "לבטל" את שאר השאילתה המקורית של האתר. אנחנו עושים את זה בעזרת סימני הערה (Comments) שגורמים ל-Database להתעלם מכל מה שבא אחרי ההזרקה שלנו.

- **ברוב המקומות:** משתמשים ב-`--` (פעמיים מקף).

- **ב-MySQL:** יש חוק מעצבן – חייב לבוא **רווח** אחרי המקפים.

- אם תכתוב `--`, זה לא יעבוד.

- אם תכתוב `--` (עם רווח בסוף), זה יעבוד.

| MySQL      | UNION SELECT NULL, NULL     | -- עם רווח |
| ---------- | --------------------------- | ---------- |
| Oracle     | UNION SELECT NULL FROM DUAL | --         |
| PostgreSQL | UNION SELECT NULL, NULL     | --         |
**טיפ ל-URL:** מכיוון שדפדפנים מוחקים רווחים בסוף הכתובת, ב-URL אנחנו כותבים `--+`

### שליפת נתונים מטבלאות אחרות #Data-Retrieval

ברגע שפיצחנו את המבנה של השאילתה המקורית (מספר עמודות וסוג נתונים), אנחנו יכולים להזריק שאילתה חדשה לגמרי שתשלוף מידע מטבלאות רגישות.

####  דוגמה מעשית:

נניח שמצאנו שהשאילתה המקורית מחזירה **2 עמודות** וששתיהן יכולות להכיל **טקסט**. אנחנו רוצים לשלוף את כל המשתמשים והסיסמאות מטבלה שנקראת `users`.

**ה-Payload:** `' UNION SELECT username, password FROM users--`

**מה קורה מאחורי הקלעים?** האתר מציג בדרך כלל רשימת מוצרים. אחרי ההזרקה, הוא יציג את רשימת המוצרים **ובנוסף** שורות חדשות שבהן במקום "שם מוצר" יופיע `username` ובמקום "תיאור מוצר" תופיע ה-`password`.

---

### ❓ הבעיה: איך יודעים איך קוראים לטבלה?

בדוגמה למעלה הנחנו שהטבלה נקראת `users` והעמודות הן `username` ו-`password`. אבל בחיים האמיתיים זה לא תמיד ככה (לפעמים זה `accounts`, `admin_users` וכו').

**איך מגלים את השמות?**

1. **ניחוש מושכל:** הרבה מערכות משתמשות בשמות סטנדרטיים.

2. **Database Enumeration (השיטה המדעית):** לכל מסד נתונים מודרני יש טבלאות מערכת ששומרות את ה"מפה" של כל ה-Database.

- ב-**MySQL/PostgreSQL:** משתמשים ב-`information_schema.tables`.
  
- ב-**Oracle:** משתמשים ב-`all_tables`.
